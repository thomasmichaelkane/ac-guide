ao_cropper

Crop and scale subsections of large AOSLO retinal images — with automatic coordinate extraction in degrees and microns.

Python 3.12+ MIT license tkinter GUI uv

About #

ao_cropper is a desktop tool for efficiently cropping and scaling sections of large AOSLO retinal images. It provides a maps-style zoomable canvas where you place a foveal centre point and an unlimited number of crop boxes with a double-click. All crop coordinates are automatically computed in degrees and microns relative to the foveal centre.

On save, labelled crop TIFFs, annotated overview maps, and a CSV of crop metadata are written for every imaging modality found in the same folder as the input file. Multiple modalities (e.g. confocal, split detector) are detected automatically and cropped simultaneously from a single session.

Maps-style canvas

Zoom and pan a very large TIFF without loading the whole image into memory at once — only the visible tile is rendered.

Automatic coordinates

Each crop's position is calculated in pixels, microns, and degrees relative to the foveal centre using the Drasdo–Fowler model eye.

Multi-modality

All TIFFs sharing the same filename base are cropped at the same time. One session produces crops for every modality.

Locate mode

Save annotated overview maps only — useful for reviewing crop placement before committing to a full extraction run.

Installation #

ao_cropper uses uv for package and environment management. You do not need to manage a virtualenv manually.

  1. Install uv

    Follow the instructions on the uv installation page for your operating system.

  2. Clone the repository 
    git clone https://github.com/thomasmichaelkane/ao_cropper.git
cd ao_cropper
  3. Install dependencies 
    uv sync

    This creates a .venv folder and installs all required packages in one step.

⚠

Working directory: ao_cropper must always be run from the project root directory (where config.yaml lives). Running it from another location will cause an error on startup.

Running #

uv run ao-cropper <image_path.tif> <OD|OS>

image_path must be a single-layer TIFF file. OD is the right eye (oculus dexter), OS is the left eye (oculus sinister). The eye side affects the nasal/temporal axis of the coordinate system.

Examples

# Basic run — uses defaults from config.yaml
uv run ao-cropper MM_0364_OS_combined_0p3796umpx_split.tif OS

# Override axial length and microns-per-pixel for this patient
uv run ao-cropper MM_0364_OS_combined_0p3796umpx_split.tif OS -a 23.5 -m 0.38

# Locate mode — save overview maps only, no individual crop TIFFs
uv run ao-cropper MM_0364_OS_combined_0p3796umpx_split.tif OS --locate

Preparing images #

File naming convention

Input TIFFs must follow this naming pattern so that ao_cropper can parse the subject ID and detect additional modalities automatically: 

<ID>_<other_info>_<MODALITY>.tif

The last underscore-delimited segment before .tif is treated as the modality label. All .tif files in the same folder that share the same <ID>_<other_info>_ prefix are detected as additional modalities and cropped simultaneously.

Example filenameSubject IDModality
MM_0364_OS_0p38umpx_split.tifMM_0364split
MM_0364_OS_0p38umpx_confocal.tifMM_0364confocal
MM_0364_OS_0p38umpx_avg.tifMM_0364avg

The underscores_in_id_count setting in config.yaml tells the parser how many underscores appear inside the subject ID itself. For MM_0364 the value is 1; for an ID like P001 the value would be 0.

ℹ

TIFFs must be single-layer (grayscale). Multi-page or multi-channel TIFFs are not supported.

Acquisition parameters

Two patient-specific values affect all degree calculations. Set the defaults in config.yaml and override per-run with CLI flags:

Parameterconfig.yaml keyCLI flagDescription
Axial lengthacquisition.axial_length-a / --axial-lengthPatient axial length in mm — scales the degree conversion.

Interface overview #

ao_cropper opens two windows simultaneously:

Canvas window

The main zoomable and pannable view of the TIFF image. All interaction (placing the foveal centre, placing crops, deleting crops) happens here.

Control panel

Shows subject metadata, a live list of placed crops with their ophthalmic coordinates, and the Save / Delete-all buttons.

Placing the foveal centre #

The foveal centre is the anchor for every coordinate calculation. It must be placed before any crop boxes can be added.

  1. Zoom and pan to the fovea

    Use the scroll wheel to zoom in and click-drag to pan until the foveal pit is clearly visible.

  2. Double-click on the foveal pit

    A full-length crosshair appears centred on the click point. Degree tick marks (or concentric rings) are drawn along the crosshair arms.

  3. Review placement

    If the centre is not accurate, click Move centre in the control panel to remove it and start again. Any crop boxes already placed are also removed at this point.

✓

Zoom in as far as possible before placing the foveal centre — a small error in the centre position propagates to every crop coordinate.

Placing crops #

After the foveal centre is set, every subsequent double-click places a crop box. Crop boxes are sized in microns (set via crop_size in config.yaml) and are automatically scaled to match the current zoom level.

  1. Navigate to the region of interest

    Zoom and pan to the area you want to crop.

  2. Double-click to place a crop box

    A red-outlined square appears. The crop number is shown in the top-left corner of the box. The control panel list updates immediately with the crop's coordinates.

  3. Delete a box if needed

    Right-click on a box to remove it. The remaining boxes are renumbered and the list updates.

  4. Repeat for all regions of interest

    There is no limit to the number of crops in a single session.

ℹ

Hovering over a crop box turns its outline yellow, making it easier to identify which box corresponds to which entry in the control panel list.

Locate mode #

Locate mode saves annotated overview maps and a coordinate CSV without extracting individual crop TIFFs. It is useful for reviewing placement accuracy before running a full extraction. 

uv run ao-cropper MM_0364_OS_0p38umpx_split.tif OS --locate

The output folder is suffixed with _locate:

ao_crops_2025-06-01_14-30-00_locate/
  Canvases/          # annotated overview per modality
  crop_location_data.csv
  LUT.csv
✓

Run in locate mode first on a new dataset to verify that the foveal centre and crop positions look correct before committing to a full extraction session.

Saving #

Click Save in the control panel when you are happy with all crop placements. ao_cropper creates a timestamped output folder alongside the input image and writes all files automatically, then closes.

⚠

There is currently no "save and continue" option — Save writes outputs and closes the session. Place all crops before saving.

Crop TIFFs

Each crop is saved as a separate TIFF labelled with its ophthalmic coordinates. Example filename:

MM_0364_OS_2p1S_3p4N_55μm_crop-1_confocal.tif

crop_location_data.csv

One row per crop, with the following columns:

ColumnDescription
Crop NumberInteger ID matching the label on the canvas.
CoordV (°)Vertical eccentricity in degrees from the foveal centre.
MeridianVS (superior) or I (inferior).
CoordH (°)Horizontal eccentricity in degrees.
MeridianHN (nasal) or T (temporal).
Distance (°)Euclidean distance from foveal centre in degrees.
Distance (µm)Euclidean distance in microns.
Centre Pixel (x)Absolute pixel x-coordinate of the crop centre in the original image.
Centre Pixel (y)Absolute pixel y-coordinate of the crop centre in the original image.

LUT.csv

A two-column lookup table containing the subject ID and the mpp value used for that session. Useful for downstream analysis pipelines that need to recover the scale.

Canvases

Full-resolution copies of each modality image with the crosshair and all crop boxes drawn on as coloured overlays. One canvas TIFF is saved per modality.

Configuration #

All tunable parameters live in config.yaml in the project root.

# Patient/image-specific — override per run with -a / -m flags
acquisition:
  axial_length: 24.0        # mm — affects all degree calculations
  mpp: 0.6                  # µm/px — from image acquisition metadata

units:
  crop_size: 55             # µm — size of each square crop region
  model_eye_length: 24.0    # mm — Drasdo & Fowler model eye reference
  reference_mpd: 291        # µm/° — model eye microns per degree
  max_image_pixels: 1_000_000_000

text:
  underscores_in_id_count: 1   # underscores inside the subject ID
  font_size: 48

crosshair:
  colour: white
  length_pixels: 20_000
  mark_width: 15

crop_box:
  colour: red
  hover_colour: yellow
  label_font_size: 48
KeyDescriptionTypical change
acquisition.axial_lengthDefault patient axial length (mm).Set to your cohort mean; override per patient with -a.
acquisition.mppDefault microns per pixel.Set to your system default; override per image with -m.
units.crop_sizeSide length of each crop in microns.Change for different study protocols.
text.underscores_in_id_countUnderscores inside the subject ID.Change if your ID format differs (e.g. P001 → 0).
crop_box.colourCrop box outline colour (any PIL colour name).Personal preference.

Coordinate system #

All crop distances are computed relative to the foveal centre. The degree conversion uses the Drasdo–Fowler model eye scaled by the patient's axial length and the image mpp.

Axes and meridians

AxisPositive directionMeridian label
Horizontal (x)NasalN (nasal) / T (temporal)
Vertical (y)Inferior (image y increases downward)I (inferior) / S (superior)

For OS (left) eyes, the horizontal axis is flipped so that nasal and temporal remain anatomically correct regardless of how the image is oriented on screen.

Tips & best practices #

Before each session

  • Confirm mpp and axial_length are correct for this patient — even small errors propagate to every crop coordinate.
  • Check that all modality TIFFs are in the same folder and follow the naming convention.
  • Run with --locate on a test set before a full extraction session.

During a session

  • Zoom in as far as possible before placing the foveal centre.
  • Use the Rings toggle to display concentric degree circles — helpful for placing crops at specific eccentricities.
  • Right-click to delete a misplaced box rather than saving and rerunning.

Troubleshooting #

config.yaml not found

ao_cropper must be run from the project root directory. Make sure your current working directory is the repository root before calling uv run ao-cropper ....

No additional modalities detected

Check that the other modality TIFFs are in the same folder as the input file and share the same base filename up to (but not including) the modality segment. Also verify that underscores_in_id_count in config.yaml matches your subject ID format.

Image fails to open / PIL safety limit error

Increase units.max_image_pixels in config.yaml. The default is 1_000_000_000 (1 billion pixels).

Crop coordinates look wrong

Double-check that the correct eye side (OD / OS) was passed, and that mpp and axial_length are correct for this patient.
